日本の薬局で利用する処方箋OCR補助プロンプト v1。
処方箋画像から患者情報、保険情報、医療機関情報、処方薬情報を抽出する。
不明な項目は空文字またはnullにし、推測項目はneeds_human_check=trueにする。
AI解析結果は人間確認前の仮データであり、直接QR化しない。


追加方針:
- 固定項目に入らない帳票上の項目も form_fields に列挙する。
- 空欄でも帳票上に存在する主要項目は form_fields に入れる。
- 拠点ごとに必要項目が違うため、出力対象はAIが確定せず、人間が確認画面で選択する。
- include_default は「通常出力に使いそうな項目」だけ true にする。
- 公費、保険、患者、処方、医療機関、備考、薬局記入欄、QR有無などをできるだけ拾う。

薬品名抽出の追加方針:
- 一般名、商品名、販売名、先発/後発、屋号違いをできるだけ区別する。
- 同じ処方ブロック内に商品名と一般名が併記されている場合、原則として別薬に分けず1つの処方薬として返す。
- その場合は drug_name に代表薬品名、generic_name に一般名候補、brand_name に商品名候補、raw_drug_text に処方欄で読めた薬品名関連行を改行区切りで返す。
- 同一薬か別薬か判断できない場合は name_relation="multiple_candidates" とし、needs_human_check=true にする。
- 一般名と商品名の紐づけは人間確認後に補助学習DBへ保存するため、推測で確定しない。

画面生成/項目抽出の追加方針:
- 結果確認画面は、事前に用意した患者/保険/医療機関/処方薬/その他項目の枠テンプレートにAI結果を流し込んで作る。
- AIは field_group と value_type を正しく付与し、画面側が適切な入力枠を選べるようにする。
- 画像内に見える項目は、固定項目に該当しない場合でも form_fields に残す。
- 人間が結果確認画面で修正し、その後の使用項目選択画面で「使う/使わない」を選ぶ。
- 補助学習では、AI値、人間修正後値、選択有無、信頼度、要確認フラグから項目採用スコアを作る。

動的帳票項目の強化方針:
- 画面には固定テンプレートだけでなく、AIが検出した帳票ラベルごとの空欄入力枠を動的に生成する。
- 画像内に枠やタイトルが見える項目は、値が空欄でも form_fields に含める。
- 空欄の枠は value="", is_empty_cell=true, ui_template="blank_cell" または適切な入力型にする。
- ui_template は input / textarea / date / number / select / checkbox / drug_line / blank_cell / unknown のいずれかにする。
- display_order は画面に並べる順序として、帳票上の上から下、左から右の順に付ける。
- 「薬品名元テキスト」は出力に使う項目ではなく、補助学習用の証跡として扱う。
- 使用項目の選択画面では、QRや後続出力に使う候補だけを選ばせ、薬品名元テキストや一般名・商品名の関係は補助学習データとして別保存する。

一般名処方マスタ連携方針:
- サーバー上のPDFをOpenAI APIが直接参照するわけではない。
- PDFから抽出した一般名処方マスタJSONをサーバー側で参照し、一般名候補の補助判定に使う。
- 一般名候補と商品名候補の紐づけは自動確定せず、人間確認後に補助学習DBへ保存する。
