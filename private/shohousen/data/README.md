# 処方箋OCR補助学習用 薬品・一般名・HOT9 照合辞書

## 元データ

- `y_r07_ALL20260317.csv`: 薬品マスタ。商品名/販売名、YJコード、一般名コード、一般名を含む。
- `h20260531/MEDIS20260531_HOT9.TXT`: HOT9マスタ。HOT9/HOT7、薬価基準収載医薬品コード、個別医薬品コード、レセ電コード、製造/販売会社を含む。
- `ippanmeishohoumaster_260612.xlsx`: 一般名処方マスタ。一般名コード、標準的な一般名記載、成分名、規格、例外コード表を含む。

## 作成ファイル

- `generic_groups.json`  
  一般名コード単位に、紐づく商品名/YJ/HOT9をまとめたメイン辞書。

- `generic_prefix8_index.json`  
  一般名コード先頭8桁から一般名候補を引くための索引。prefix8だけでは粗いので、自動確定には使わず候補抽出用。

- `product_index.jsonl`  
  商品/販売名単位の検索用マスタ。1行1商品。OCRで読んだ薬品名を候補検索する時に使う。

- `relation_edges.jsonl`  
  商品名/YJコードと一般名コードの関係。`confidence` と `match_type` を持つ。

- `aliases.jsonl`  
  薬品名、表示名、カナ、一般名、成分名から作った検索別名辞書。`alias_normalized` でOCR文字列を照合する。

- `ippan_exception_products.jsonl`  
  一般名処方マスタの例外コード品目対照表。

- `unmatched_report.csv`  
  完全一致しなかった関係、マスタに片方だけあるコードの確認用。

- `summary.json`  
  件数、照合結果、推奨コード運用ルール。

## 照合結果サマリ

- y_r07薬品件数: 19,337
- 一般名コードあり: 8,751
- y_r07上の一般名コード種類: 1,381
- HOT9件数: 38,362
- 一般名処方マスタの一般名コード: 1,230
- 一般名処方マスタの例外品目: 66

## 重要なコード照合ルール

実データを確認した結果、一般名コードと商品側YJコードは次の順で照合するのが安全です。

1. y_r07に一般名コードがある場合は、それを最優先する。
2. YJコードから推定する場合は `YJコード先頭9桁 + ZZZ` を一般名コード候補にする。
3. `AZZZ / BZZZ / CZZZ / DZZZ` のような一般名コードは、単純な先頭9桁一致ではなく、例外・集約グループとして扱う。
4. 先頭8桁一致は候補抽出には使えるが、自動確定には使わない。

今回のy_r07では、一般名コードあり 8,751 件のうち、`YJ先頭9桁 + ZZZ` の完全一致は 8,634 件でした。  
残り 107 件は、主に `AZZZ/BZZZ/CZZZ` の例外・集約系です。

## 処方箋OCRでの使い方

1. OCRで読んだ薬品名をNFKC正規化する。
2. `aliases.jsonl.alias_normalized` に部分一致/類似一致をかける。
3. 候補の `product_index` と `relation_edges` から一般名候補を出す。
4. `confidence=high` は強い候補、`medium_exception_or_aggregate` は要確認候補として出す。
5. 人間が確定した内容を補助学習DBに保存し、次回以降の優先度に反映する。
